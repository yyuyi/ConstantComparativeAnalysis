CCA 简化 Evaluation Metrics

这个版本把原来 20 多个细项压缩为 5 个核心维度。目标不是证明每一个 qualitative 判断都完美，而是判断 CCA 程序是否真的可用，以及它和人类文章分析相比是否足够接近、可追溯、可修正。

核心问题 1程序有没有完整、稳定、可审计地跑完 CCA？
核心问题 2结果是否被 transcript 和 quote 支撑，而不是看起来像主题总结？
核心问题 3和人类分析相比，核心发现是否对齐，偏差在哪里？

最终评分结构

总分使用 1-5 分，也可转换为 100 分。建议阈值：4.0 以上 = 程序基本 work；3.5-3.9 = 可用于开发但需要修正；低于 3.5 = 不能说明程序稳定有效。若“证据可追溯”低于 3.0，即使总分较高，也不建议通过。

机器评估部分

机器部分只负责可重复、可自动审计的检查。它不能替代人类 judgment，但可以减少人类评分者需要看的原始材料数量。

检查项 需要看的数据 机器要输出什么 失败信号

人类评分部分

人类评分者不需要读完所有 270 个 segment。推荐使用“全量高层输出 + 抽样原始数据”的方式，重点判断机器结果是否能站得住。

人类评分者必须阅读

  1. 人类文章的 Results、Discussion、Conclusion，以及任何 themes/table/codebook 描述。
  2. 机器输出的 integrated_cca_summary.txt、integrated_categories.txt、integrated_incident_patterns.txt。
  3. 两个 coder 的 category_comparisons_*.txt，重点看每个 category 的 supporting segments 和 quotes。
  4. 机器生成的 process/quote audit 表，如果存在，例如 process_metrics.json、quote_verification.json。

人类评分者抽样查看的原始数据

  • 所有最终 category quotes：如果超过 30 条，则每个 category 至少 3 条，总数控制在 30 条左右。
  • 20 个 transcript segments：10 个来自 final categories 的 supporting segments，5 个随机 uncited segments，5 个边界/分歧/长文本/低置信度 segments。
  • 如果数据集少于 50 个 segments，建议全看；50-500 个 segments，抽 20-30 个；超过 500 个 segments，抽 30-40 个并按 category 分层。
建议两名人类评分者独立评分。 若任一维度分差达到 1.0 分或以上，需要讨论并记录 adjudicated score。若只有一名评分者，报告中要标记为 single-rater evaluation。

推荐评分流程

  1. 机器先跑完整性和 quote audit。 输出 file manifest、coverage counts、quote verification、quote quality flags。
  2. 人类先建立 article reference checklist。 从 published article 的 Results/Discussion/Conclusion 提取 8-15 个 expected themes，不要求机器复刻文章结构，但核心 substantive findings 应该被覆盖。
  3. 人类阅读机器高层输出。 看 integrated summary、integrated categories、incident patterns、category comparisons。
  4. 人类做 transcript evidence sample audit。 检查所有 final quotes 或最多 30 条 quote，再检查 20-30 个 segments 的 category fit、missed theme、comparison quality。
  5. 按 5 个维度评分。 每个维度给 1-5 分和一句 rationale；两个评分者取均值或 adjudicated score。
  6. 形成 go/no-go 判断。 4.0+ 说明程序基本 work；3.5-3.9 说明可开发但需修正；低于 3.5 或证据维度低于 3.0 说明不能通过。

最小输出模板

每次 evaluation 至少应输出以下内容，避免报告过重但保留必要可审计性。

模块 必须包含
Verdict 程序是否 work；是否接近人类分析；是否可用于下一轮开发/evaluation
Score table 5 个维度的 score、weight、rationale、next fix
Human theme comparison human article theme checklist：strong / partial / missing / extra unsupported
Evidence audit quote verification summary、sampled segment audit summary、主要 evidence 风险
Fix list 最多 5 个优先修正点，直接对应程序开发
