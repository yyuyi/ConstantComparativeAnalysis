CCA Agent 方法学流程图

当前 pipeline 的核心新意是把 Constant Comparative Analysis（CCA，持续比较分析）拆成可审计的计算步骤：从 transcript segment 中识别 incident，给出 incident label，比较相似与差异，再通过全数据记忆、全局 reconciliation、证据校验和反思性评估形成可追溯的 comparative synthesis。

方法 = CCA | 基本单位 = Incident | 目标 = 可追溯的比较性解释

Key Concepts / 关键术语解释

Transcript Segment / 文本片段 系统把访谈 transcript 切成较小片段。每个片段是后续 coding 和 comparison 的输入单位，不等同于完整访谈。
Incident / 事件性分析单位 CCA 不是先找大主题，而是先识别片段里发生了什么、表达了什么行动、经验、条件、判断或过程。这个可被比较的分析单位就是 incident。
Incident Label / 事件标签 对 incident 的 2-6 个词概括，例如 “child safety customization”。它不是关键词摘录，而是分析性短标签，用来支持后续相似/差异比较。
Comparison Notes / 比较笔记 记录当前 incident 与先前 incidents 的 similarities 和 differences。CCA 的核心不只是编码，而是持续比较。
Analytic Memo / 分析备忘录 记录为什么需要新增、合并或细化 label/category，以及还存在什么 tension、疑问或 sampling lead。
Comparative Category / 比较性类别 由多个 incidents 通过比较聚合而来。好的 category 要有 defining properties、内部 variation、与其他 category 的边界和关系。
Boundary / Negative Case Boundary case 是模糊边界案例；negative case 是挑战现有解释的反例。CCA 需要它们来避免过早形成单一结论。
Reconciliation / 全局再协调 第一轮 incident coding 后，系统再看全数据，合并近义 labels，找跨 transcript 和早晚片段之间的比较，补足局部窗口遗漏。
Traceability / 可追溯性 最终 claim 必须能追溯回 category、memo、incident、segment 和 quote。它是机器辅助 qualitative analysis 最重要的可信度基础之一。

End-to-End Workflow

01

Run Setup

读取参数、API key、analysis mode、segment length、coder 数和 CCA memory/evaluation 设置。这一步决定模型能力、片段大小、是否开启 CAC、是否启用全局 reconciliation 和 reflective evaluation。

02

Transcript Ingestion

读取 txt/pdf/docx，切分 transcript segment，并建立 Chroma vector index。这里的 vector index 之后用于检索相关片段和证据 quote。

03

Transcript Summaries

为每份 transcript 生成 summary，作为后续 category/memo/synthesis 的背景上下文。summary 不替代原文，只帮助模型保持访谈整体语境。

04

Hybrid Incident Memory

每个 segment 看到 recent prior、retrieved prior、anchor prior。这样既保留局部连续性，也把早期、远处、边界案例带回来，减少 sliding-window 只看近期片段的问题。

05

Incident Coding

为每个 segment 生成 2-6 word incident labels，并写出 similarities/differences 和 analytic memo。这里评估重点是 label 是否贴合 segment，以及比较是否具体。

06

Global Reconciliation

第二轮全数据比较：合并近义 labels，标记 early-late、cross-transcript、boundary/negative cases。这是对 collaborator 担心的 sliding-window 局限的直接回应。

07

Category Comparison

从 reconciled incidents 建立 comparative categories、defining properties、category contrasts。这里不是传统 axial coding，而是从 incident 比较推进到 category 比较。

08

Quote Retrieval & Verification

用 vector index 检索证据 quote；quote 必须能在 retrieved context 中校验，否则回退到 supporting segment。这一步把 category claim 和原始 transcript 连接起来。

09

Comparative Memoing

生成 3-6 个 memos，记录 tensions、unanswered questions、sampling leads。memo 用来展示分析如何被比较推进，而不是只给最终答案。

10

CCA Synthesis

整合 incidents、categories、memos，输出单段 comparative narrative。好的 synthesis 应解释 category 之间如何互相强化、冲突、限定或转化。

11

Cross-Coder Integration

两位 coder 时整合 incident patterns、category matrix、memo digest、unified summary。重点是区分 convergence 和 divergence，而不是简单平均。

12

Reflective Evaluation

独立 audit agent 评分 process fidelity、traceability、comparison coverage、coherence。它不改写结果，只做质量审计。

Hybrid Memory Design

Recent Prior

  • 默认最近 20 个 incidents。
  • 保留 sequential analytic continuity。
  • 适合捕捉同一 transcript 内逐步细化的比较。

Retrieved Prior

  • 从历史 archive 中按 token overlap 检索相关 incident。
  • 让新片段能和更早的相似/相反案例比较。
  • 解决单纯 sliding window 遗忘早期 context 的问题。

Anchor Prior

  • 保留早期和分布式 anchor cases。
  • 帮助模型维持 category 的历史边界。
  • 为 boundary/negative case comparison 提供稳定参照。

Evaluation Matrix

CCA-Specific Quality Dimensions

Comparison Fidelity 是否真正比较 incidents，而不是只生成 labels。
Coverage 是否覆盖 early/late、cross-transcript、anchor 和 retrieved cases。
Variation Handling 是否处理 boundary cases、negative cases 和条件差异。
Auditability claim 是否可追溯到 segment、quote、memo 和 comparison。

Retained Qualitative Standards

Grounding categories 和 synthesis 是否有原文证据支持。
Coherence 最终 narrative 是否连贯并解释 category 关系。
Non-Redundancy categories 是否边界清楚、避免重复。
Article Resonance 与原文章 findings 有合理呼应，但不强制复制 GT theory。

Concrete Evaluation Protocol / 具体怎么评估

你需要准备什么

因为你有原文 paper 和原始 transcript，可以做比单纯机器评估更可信的 hybrid evaluation。

  1. 原始 transcripts：用于判断 incident labels、comparison notes、quotes 是否贴合数据。
  2. Published papers：抽取 published themes、findings、theoretical relationships，用于外部 plausibility 对照。
  3. Agent outputs：incident coding、reconciliation、categories、memos、synthesis、evaluation files。
  4. 人工评分表：按 rubric 逐条打 1-5 分，并写一句理由。

机器先做什么

机器适合做可程序化、可审计、低解释负担的检查。

  • 检查输出文件是否存在、JSON 字段是否完整。
  • 统计 segments、incident notes、labels、categories、memos。
  • 检查每个 incident 是否有 labels、comparison notes、analytic memo。
  • 检查 quote 是否真实出现在 retrieved context 或 source segment。
  • 检查 synthesis claim 是否能追溯到 categories 和 supporting segments。

人需要判断什么

人主要判断 substantive validity 和 interpretive adequacy，避免“机器评估机器”的循环问题。

  • 抽样读 transcript segment，看 incident label 是否真的抓住意义。
  • 判断 similarities/differences 是否是实质比较，而不是模板话。
  • 判断 category 是否有解释力，是否不是普通主题列表。
  • 对照 paper findings，看 agent 是否覆盖主要发现，是否有合理的新组织方式。

Rubric Items: Who Evaluates What?

Rubric 项目 具体要做什么 主要材料 建议评估者
A1 Incident Label Fit 随机抽每篇 paper 的 20-30 个 transcript segments，对照 agent incident labels，判断 label 是否具体、贴合、不是泛泛主题。 Transcript + incident_coding 人评为主；机器只能检查 label 是否存在和长度。
A2 Similarity/Difference Comparison 检查 comparison_notes 是否明确写出 similarities 和 differences，并判断它们是否真的基于 prior incidents。 incident_coding + transcript sample 人机结合；机器查字段，人判断质量。
A3 Full-Dataset Coverage 看 comparison 是否覆盖 early/late、cross-transcript、retrieved prior、anchor prior，而不是只看相邻片段。 incident_coding + incident_reconciliation + logs/counts 机器可先统计；人抽查充分性。
A4 Boundary/Negative Case Handling 找输出中标记的 boundary_or_negative_case，回读原 segment，判断它是否真的挑战或限定了 category。 incident_reconciliation + transcript 人评为主。
B1 Category Emergence 从 category supporting_segments 回看 incident labels 和 memos，判断 category 是否由 incidents 比较形成。 category_comparisons + incident_coding + memos 人机结合。
B2 Category Boundary Clarity 比较 categories 之间是否重复，是否有清楚 defining properties 和 contrasts。 category_comparisons 人评为主；机器可辅助找相似 category names。
B3 Properties and Dimensions 检查每个 category 是否包含 properties/dimensions，并能解释内部 variation。 category_comparisons + memos 人评为主。
B4 Category-Level Comparative Insight 判断 category 描述是否解释 overlap、contrast、condition、relationship，而不是只列主题。 category_comparisons 人评为主。
C1 Segment Traceability 检查每个 category 和主要 claim 是否有 transcript + segment number 支撑。 category_comparisons + synthesis 机器为主；人抽查。
C2 Quote Verification 确认 supporting_quotes 是原文子串，并判断 quote 是否支持对应 category claim。 Transcript + quote_audit + category_comparisons 机器验证原文存在；人判断支持力度。
C3 Claim-to-Comparison Audit Trail 从 final synthesis 的每个 major claim 往回追到 category、memo、incident、segment。 synthesis + categories + memos + incidents 人机结合；最终人评。
D1 Memo Use for Refinement 判断 memo 是否真正记录 refinement、tensions、sampling leads，而不是重复 category。 cca_memos 人评为主。
D2 Tension Preservation 看 unresolved tensions 是否被保留并进入 synthesis 或 next steps，而不是被强行消解。 memos + synthesis 人评为主。
E1 Comparative Narrative Coherence 读最终单段 synthesis，判断它是否解释 category 关系，而不是罗列主题。 cca_synthesis / integrated_cca_summary 人评为主。
E2 Variation-Aware Explanation 看 synthesis 是否说明不同 contexts、conditions、actors 或 processes 如何改变 category 关系。 synthesis + categories + transcript sample 人评为主。
E3 Published-Article Resonance 从原 paper 抽取 major findings/themes/relationships，比较 agent 是否覆盖核心发现，同时允许 CCA 有不同组织方式。 Published paper + agent synthesis/categories 人评为主；不要用 paper incident label 对答案。
F1 Cross-Coder Convergence 检查 integrated_incident_patterns 和 integrated_categories 是否合并相近 coder outputs，而不是一 label 一个 pattern。 integrated outputs + per-coder outputs 机器可统计数量；人判断合并质量。
F2 Cross-Coder Divergence 判断系统是否保留 coder 差异，并把差异用于 refining category boundaries 或 tensions。 integrated_memo_digest + integrated_categories 人评为主。
G1/G2 Process Fidelity 检查文件完整性、schema 完整性、count accuracy，特别区分 segments、incident notes、incident labels。 All output files + analysis_summary 机器为主。

Paper Contribution Logic

这套方法可以写成区别于 GT agent 的 CCA paper：同样数据集下，贡献不再是自动生成 grounded theory，而是展示 LLM 如何在 context-window 限制下维持 full-dataset constant comparison，并用 hybrid memory、global reconciliation 和 reflective evaluation 把比较过程变成可审计的方法学对象。
